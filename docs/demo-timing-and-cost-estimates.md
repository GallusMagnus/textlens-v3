# TextLens Demo Timing And Cost Estimates

Last updated: 2026-06-23

## Purpose

This note records a practical demo-phase rule of thumb for TextLens analysis timing and API cost.

These figures are provisional. They are based on measured runs from a longer article and should be refined with more sample analyses across short, medium, and long texts.

## Current Working Rules

- Consumer Mode is the faster and cheaper path.
- Professional Modes are currently about 3x slower and about 3x more costly than Consumer Mode for a longer article.
- `gpt-5.4` is roughly half the API cost of `gpt-5.5`.
- Cost scales with article length and with how many staged calls are triggered.

## Measured Example

Article used for comparison:

- Length: 2,416 words
- Mode family comparison:
  - Consumer Mode
  - Consensus Standards Mode (professional mode family)

### Measured Timing

| Run Type | Model | Total Time | Approx. Minutes |
| --- | --- | ---: | ---: |
| Consumer Mode | `gpt-5.5` | 92.28s | 1.54 min |
| Consensus Standards Mode | `gpt-5.5` | 285.87s | 4.76 min |

### Timing Ratio

| Comparison | Result |
| --- | --- |
| Professional vs Consumer | about 3.1x slower |

## Why Professional Modes Cost More

Consumer Mode currently makes:

- 1 metadata extraction call
- 1 analysis call

Professional modes can make:

- 1 metadata extraction call
- multiple guardrail calls
- multiple classification calls
- 1 synthesis call

In the measured professional example, the system made:

- 1 metadata extraction call
- 3 guardrail calls
- 3 classification calls
- 1 synthesis call

This is the main reason professional modes take materially longer and cost materially more.

## Pricing Basis

Working pricing reference:

- `gpt-5.4`: about `$2.50 / 1M input tokens` and `$15.00 / 1M output tokens`
- `gpt-5.5`: about `$5.00 / 1M input tokens` and `$30.00 / 1M output tokens`

Reference: OpenAI API pricing documentation  
<https://developers.openai.com/api/docs/pricing>

### Rule-Of-Thumb By Article Length

| Article Length | Consumer Mode on `gpt-5.4` | Professional Modes on `gpt-5.4` | Notes |
| --- | ---: | ---: | --- |
| Short: under 1,000 words | `$0.02-$0.05` | `$0.07-$0.12` | Usually lighter prompt and passage load |
| Medium: 1,000-2,000 words | `$0.03-$0.06` | `$0.10-$0.18` | Typical opinion/editorial length |
| Long: 2,000-3,000 words | `$0.05-$0.08` | `$0.15-$0.25` | The measured example falls here |
| Heavy: over 3,000 words | `$0.06-$0.10+` | `$0.20-$0.40+` | More likely to trigger maximum staged batching |

## Suggested Demo Budget Framing

For the public `gpt-5.4` deployment:

| Volume | Mostly Consumer Usage | Mostly Professional Usage | Mixed Usage |
| --- | ---: | ---: | ---: |
| 25 analyses | `$1-$2` | `$4-$6` | `$3-$4` |
| 50 analyses | `$2-$4` | `$8-$12` | `$5-$8` |
| 100 analyses | `$4-$8` | `$15-$25` | `$10-$16` |

These are intentionally conservative planning figures for a demo phase rather than a production finance model.

## Confidence And Caveats

- These are estimates derived from measured runtime and prompt structure, not exact token usage logs.
- Actual billed cost may vary depending on:
  - the number of passages selected for staged review
  - the amount of output generated by the model
  - prompt caching effects
  - future prompt or ruleset changes
  - future OpenAI pricing changes

## Practical Conclusion

- Consumer Mode is suitable for fast, lower-cost exploratory review.
- Professional Modes are suitable for slower, more rigorous standards-based audit.
- For current demo planning, a useful shorthand is:
  - Professional Mode: about 3x longer and 3x more expensive than Consumer Mode
  - `gpt-5.4`: about half the cost of `gpt-5.5`
